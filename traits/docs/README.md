# Traits Project Documentation

This project teaches Rust traits through small, runnable examples.

Traits are Rust's way to describe shared behavior. A trait says what a type can
do, and each type decides how it does that behavior.

## What This Project Covers

- Defining a trait and implementing it for multiple structs
- Default trait methods
- Trait bounds on generic functions
- `impl Trait` parameters
- Real-world patterns: validators, repositories, services, strategies, and formatters
- Trait objects with `dyn Trait`
- Associated types
- Supertraits
- `where` clauses
- Operator traits
- Derived traits such as `Debug`, `Clone`, `PartialEq`, and `Default`

## How To Run

```powershell
cargo run
```

## Documentation Style

Each documentation file explains what the example is used for, why that pattern
is chosen, the advantages, disadvantages, and what to notice in the code.
